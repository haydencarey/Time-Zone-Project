Documentation


I was first drawn to a creative experience that can visualize who is awake and who is asleep. Although in the end, I strayed away from a more creative visualization of data, I'm happy I developed a functional web app that tells the user helpful information. The best way to describe my project is through a series of user experiences:

  1. I found an API that requires latitude and longitude coordinates. I first wanted the user to input the coordinates of a desired location. I created some "label" elements, allowed the user to manipulate the API with template literals (${}), and followed Mathura’s Pokemon video on how to access an API. I set parameters, for three returns:
    • Between 8 am and 6 pm: Awake and Working Hours
    • Between 6 pm and 11 pm: Awake, but Outside Working Hours
    • Between 11 pm and 8 am: Asleep


  2. So I was proud of that, but my project still felt like it had more potential. I met with Ellen during office hours and she suggested eliminating some of the work done by the user. We came up with a plan of implenting another JSON data set; one made by me. It would be a list of selected cities with their latitude and longitude coordinates. I used MDN to create a basic dropdown with a list of cities. The challenge here was accessing my new JSON file so when the user selects one of the cities in the dropdown, this accesses the JSON data. Ellen sent me one of her P5.js sketches where she did something very similar. Referencing this, I accessed the DOM, and used this MDN reference to refer to my other JSON file. Populating the JSON data into my input boxes was the biggest challenge. To be frank, I still don’t understand. My sister (a full-stack engineer) and Aiden helped me out with that part.


  3. Ok, I was on my way to more seamless user experience. I understood that there needed to be an embedded P5.js sketch. Coming up with an idea of how to visualize my experience on a p5 canvas was a challenge. Luckily, I stumbled upon a Dan Shiffman video , where he visualizes his subscribers on a map in a p5 canvas. He inspired me to create a UX where users can click on a map, and they will be able to know if someone would be awake or asleep at that selected location. Following the Schiffman video, I imported this excellent library called Mappa.js to my HTML. Mappa.js allows you to overlay a canvas on top of a tile map. It also provides some helpful commands. It took me a whole afternoon to import a map due to version issues and problems with the library link. Schiffman taught me how to make an ellipse at the inputted latitude, longitude. Mappa.js has a very convenient pixelToLatLng function to convert pixels to lat and long. Lines 159 to 170 convert the pixels lat/long to and populates it in my input boxes.


  4. Potential next steps would be coming up with a more clever way to visualize the map based on who’s awake and who’s asleep. I was thinking of a dynamic map that looks something like this. One more step in the UX progression would be to allows users to open web page, look at the map, and immediately realize what part of the world is asleep and awake, instead of having to click on the map.
